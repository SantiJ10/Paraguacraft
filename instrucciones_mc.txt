CONTEXTO DEL SISTEMA Y REGLAS ESTRICTAS (NO IGNORAR):

1. Foco en Hardware Viejo (Cliente): Asume que el usuario final puede tener poca RAM y GPUs antiguas. Optimiza el consumo de memoria en el launcher, sugiere fallbacks (ej. Mesa3D) y perfiles de Java ligeros para garantizar jugabilidad universal sin importar la gama de la PC.
2. Rendimiento Sostenido 4hs+ (Servidor): El servidor y el cliente deben estar blindados contra Memory Leaks. Sugiere argumentos de JVM optimizados (ej. ZGC/G1GC tuneados) para evitar "lagazos" y caídas de TPS tras 3-4 horas de juego continuo.
3. Arquitectura Non-blocking UI: En el launcher, NUNCA bloquees el Event Loop principal. Todo I/O (descargas de mods/skins, APIs, subprocesos) debe ser asíncrono/multihilo.
4. Infraestructura desde Cero (Oracle + Linux): Todo comando de terminal debe ser para Ubuntu ARM. Da el paso a paso exacto para instalar dependencias (Java 21, tmux, playit) y transferir archivos.
5. Integración Bot-Servidor: El bot de Discord inyecta comandos a la consola de Minecraft usando Python (`os.system("tmux send-keys...")`). Esto debe ser seguro y asíncrono.
6. Gestión de Procesos y Cero Crasheos: Todo subproceso (Java, Playit) debe destruirse limpiamente al cerrar. Ante caídas de red o fallos, captura el error silenciosamente (Try/Except) y avisa a la UI.
7. Edición Segura de Archivos: La modificación de JSONs/Properties (ej. ops.json) debe ser bidireccional, permitiendo AGREGAR y REMOVER datos sin corromper la estructura.
